# 동료 협업용 콘텐츠 저작 가이드

이 문서는 **AI·디지털 교육자료 수업 지원센터(ai-dt.net)** 사이트의 콘텐츠(발행사 답변, 자주 묻는 질문(FAQ), 일반 페이지 등)를 직접 편집하고 수정하려는 동료 운영자 분들을 위한 작성 가이드입니다. 

---

## 1. 콘텐츠 협업 워크플로우

개발자가 아닌 동료 운영자는 다음 단계를 통해 사이트 콘텐츠를 발행할 수 있습니다.

1. **파일 작성**: 이 가이드를 참고하여 마크다운(`.md` 또는 `.mdx`) 파일을 생성하거나 수정합니다.
2. **운영자 전달**: 수정된 텍스트 파일과 관련 이미지 자산을 모아 메일 또는 구글 드라이브 등으로 운영자에게 보냅니다.
3. **배포**: 운영자가 해당 파일을 프로젝트 경로에 넣고 Git에 푸시하면, 수분 내로 **[ai-dt-net.pages.dev](https://ai-dt-net.pages.dev/)**에 자동으로 반영됩니다.

---

## 2. 콘텐츠 폴더 및 파일 작성 규칙

각 콘텐츠 카테고리에 맞는 폴더 경로에 텍스트 파일을 추가하거나 수정해야 합니다.

### ① 발행사 답변 (`src/content/publishers/`)
발행사(출판사) 답변 내용은 `.mdx` 파일 데이터로 구성되며, 이 데이터를 통해 상세 페이지가 자동으로 생성됩니다.
- **파일명 규칙**: `{영문이름}.mdx` (예: `cheonjae.mdx`, `visang.mdx`)
- **Frontmatter (메타 데이터) 규격**:
  ```yaml
  ---
  name: "천재교과서"      # 사이트에 노출되는 정식 발행사명
  order: 1              # 메인/허브 목록에서의 정렬 순서 (낮을수록 앞)
  sections:             # 답변 내용이 수록된 파일 섹션 매핑
    - label: "중등 수학" # 교과명
      dir: "1"          # 해당 교과 답변 이미지가 위치한 폴더 번호 (replies/{slug}/{dir}/)
    - label: "초등 정보"
      dir: "2"
  ---
  ```

### ② 자주 묻는 질문 (`src/content/faq/`)
선도학교 운영 관련 FAQ 등의 리스트를 관리합니다.
- **파일명 규칙**: `{영문이름-또는-숫자}.md` (예: `faq-01.md`)
- **Frontmatter (메타 데이터) 규격**:
  ```yaml
  ---
  q: "디지털교과서 연동 오류 시 조치 방법은 무엇인가요?" # 질문 텍스트
  category: "일반"                                  # 분류 카테고리 (필요시)
  order: 1                                         # 목록 정렬 순서 (낮을수록 앞)
  ---
  질문에 대한 상세한 답변 내용을 이곳에 마크다운 형식으로 편하게 작성합니다.
  ```

### ③ 자료 카드 및 다운로드 (`src/content/resources/`)
사전 준비사항 가이드, 소프트웨어 다운로드, 도움자료 카드 데이터를 관리합니다.
- **파일명 규칙**: `{카테고리-이름}.md` (예: `neis-guide.md`)
- **Frontmatter (메타 데이터) 규격**:
  ```yaml
  ---
  title: "나이스 연동 매뉴얼 (PDF)"        # 자료 제목
  category: "manual"                     # 카테고리 (manual, software, aidt 등)
  target: "교사"                         # 대상 분류 (교사, 학생, 공통 중 택일)
  url: "https://example.com/download"    # (선택) 외부 다운로드 링크가 있을 때 입력
  thumb: "neis-thumb.png"                # (선택) 썸네일 이미지 이름
  desc: "나이스 자료 연동을 위한 매뉴얼입니다." # 간략한 한 줄 설명
  order: 5                               # 목록 정렬 순서
  ---
  ```

### ④ 일반 위지윅형 페이지 (`src/content/pages/`)
홈페이지, 서비스 안내 등 일반적인 긴 문장 중심의 설명 페이지입니다.
- **파일명 규칙**: `{페이지명}.md` (예: `about.md`)
- **Frontmatter (메타 데이터) 규격**:
  ```yaml
  ---
  title: "스스로 학습 안내"        # 페이지 제목
  section: "안내"               # 대분류
  order: 3                      # 정렬 순서
  ---
  ## 1. 개요
  본문 내용을 작성합니다.
  ```

## 3. 이미지 자산 및 스크린샷 배치 규칙

### ① 발행사 답변 이미지
발행사의 답변 문서 이미지(스캔본/스크린샷)는 다음 경로에 순서대로 보관되어야 합니다.
- **경로**: `src/assets/replies/{발행사-영문-slug}/{섹션-dir-번호}/`
- **파일명**: `01.png`, `02.png`, `03.png` ... 와 같이 두 자리 숫자 형식을 권장합니다.
- **주의**: 발행사 `.mdx` 파일 내 `sections.dir` 값과 폴더명이 정확히 일치해야 합니다.

### ② 일반 페이지 인증 스크린샷 (수동 배치 필요)
학교 구글 계정 등으로 인해 이관되지 못한 특정 페이지의 매뉴얼 이미지는 다음 경로에 순서대로 넣어두면 자동으로 페이지에 노출됩니다.
- **경로 및 해당 위치**:
  - `src/assets/pages/manual/1-neis/` : 나이스 자료 연동 단계
  - `src/assets/pages/manual/2-signup/` : 교육디지털원패스 회원가입 안내
  - `src/assets/pages/eduonepass/signup/` : 교육디지털원패스 가입 방법 단계
- **규칙**: 파일명은 반드시 **`01.png`**, **`02.png`** 등 두 자리 숫자로 구성합니다 (png, jpg, webp 지원). 파일이 없으면 사이트에서 "이미지 준비 중" 안내 박스가 기본 노출됩니다.

## 4. 본문(Markdown) 작성 팁

- **제목 크기 구성**: 본문 내에서 가장 큰 대제목은 `##` (H2), 그 하위는 `###` (H3)으로 구성하여 구조화합니다.
- **줄바꿈**: 마크다운은 단순히 Enter 키 한 번으로는 줄바꿈이 되지 않습니다. 문단을 나눌 때는 Enter 키를 두 번 입력하거나, 한 줄에서 개행하려면 문장 끝에 공백(Space)을 두 번 입력합니다.
- **강조**: **굵게 표시**하려면 `**텍스트**`를 사용하고, *기울임*은 `*텍스트*`를 사용합니다.
- **링크 연결**: `[표시할 글자](연결할 주소)` 형식으로 삽입합니다. (예: `[NEIS 바로가기](https://neisplus.kr)`)
